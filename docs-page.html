Introduction Last updated: 2021-01-03

Welcome to the new question and answer bank management app. This app is designed to make managing the CET Training Program as simple as possible while maintaining compliance with approved written procedures provided by Microsoft.

Some of the capabilities of this app are as follows:

  • Add new questions to the bank.
  • Change or update existing questions. In the spirit of the Microsoft recommended quarterly review.
  • Generate a couple different kinds of random exam.
  • Create curated exams for practice or upgrade purposes.
  • Generate a random scheduled quiz question that can be sent to the team.

This help file will assist you in learning how to best utilize this app, as well as, understanding the reasons behind all the features and when/how you might want to use them.

Exam Generation

Automatic exam generation, designed to simultaneously allow the trainers to curate exams and generate random exams that are compliant with program standards by design!

Curated exams

For the purposes of creating practice exams, the ability to select specific questions by hand and generate an exam from that selection has been included.

More specifically, you can select questions from the Home View using the check boxes. Next, using the hamburger menu, you can select the "Export Selected Exam" option.

The Export Selected Exam popup.

As the name implies, this is the popup that appears when you press the Export Selected Exam button. Next, you will fill out the file name text box with any file name that you desire and press Export. The practice test and its answer key will then be downloaded as described below.

Usage Tip: The curated exam feature would be perfect for practice or training. This feature would also be useful for upgrades in the event of an exam failure.

The Algorithms

There are two different automatically generated exams. I called them the Methods of Generating. They are the Pseudorandom and Seed method.

The Pseudorandom method of generating provides the user with as random of an exam as possible given a few constraints.
  • Bank Size: Obviously, the exam bank is finite, and therefore the results of the pseudorandom algorithm can only be so random.
  • Computers cannot be "Random": The limitations of the technology that we are using is another constraint. Without venturing into quantum physics, it is impossible to ask a computer to be random. Hence "pseudo" random, behind the scenes is an algorithm that does its best to simulate what humans generally perceive as random.

How it works (in general) 

While (Number of questions already selected is less than the number of questions desired) { 
	Generate a random number between 1 and n: 
		Pull the question whose ID you just randomly generated:
			if
			(
			This question is not already in the exam AND 
			This question is at the chosen CET Level OR  below if more than 50% of the exam has been created AND
			This question is the correct category if a learning objective was selected AND
			The question wasn’t used on the previous exam OR more than 50% of the exam has already been created
			)
			then (add the question to the exam)	
}

Where n is the total number of question in the exam bank.

Some of the reasoning behind this algorithm is as follows

  1. To ensure that each exam generated (given a couple of assumptions) is a requisite amount unique from any recently generated exams.
  2. In compliance with Microsoft guidance, the different levels of exam are 50% unique from the previous level by ensuring at least half the exam is at the appropriate CET level.
  3. To allow the user to generate an exam filtered by Learning Objective, if desired.
  4. To ensure that the exam is at or below than the level of certification it represents.

The aforementioned assumptions were built in to make the "recently used" flag meaningful. Specifically, the flag on a question will only change to a value of true when the Download button is pressed. This is as opposed to every single time the generate exam button is pressed.

Those assumptions are all in the case of a Pseudorandom exam as the recently used flag is ignored for a Seeded exam, and they are as follows:
  • The user will only be pressing the Download button when they are satisfied with the exam that has been generated. This is why there is a preview window. It allows you to explore your options.
  • That the download button will only be pressed at expected intervals. That is to say, exams will be needed in a predictable way, such as an individual's certification exams or biennial requalification exams.

With the design of this algorithm and these principles in mind, the Pseudorandom exam is the preferred method of generation.

The Seeded exam is much more simple, much less random, and designed such that given the same number of questions, same CET Level, and same Learning Objectives the exam will be the same.

This does rely on some other nuances. As the exam bank grows and changes, the Seeded exams will change. Without a more complicated seeding or hashing algorithm, I cannot prevent this (though, that does sound like a fun challenge). So, the true purpose of the Seeded exams is to be able to pull up a "known" exam within a reasonable proximity to the last time you pulled it up.

It is for this reason that the Answer Key that is generated along side the Seeded exams provides you with the "Exam Details," thereby allowing you to pull up the same exam later.

The Seeded exams are based off of the number of desired questions and the number of questions in the exam bank. I did some math and came up with an algorithm that will reliably generate the maximum number of unique exams from those settings, and do so in a predictable way (given the above assumptions.)

Et Voila, you have exams that can be recalled based on some settings.

Allowing you to pull up the answer key to an exam you lost, or perhaps a "level playing field" practice exam, or a quiz that you particularly appreciate... The training possibilities are endless.

The Output

One could argue that this is the meat and potatoes of the Exam Generation function, so, I will go through step-by-step how to get a physical exam in your hand.

Step 1 Step 2 Step 3 Step 4 Step 5
As always, you will start off at the
The Home View

From the Home View, you will press the Hamburger Menu and select "Generate Exam" in the upper right-hand corner.

This will take you to the Exam Designer.

The Exam Designer
page has several interfaces. We will be creating a Pseudorandom exam.

To do so you must first select Pseudorandom in the Method of Generating dropdown box. Then you will choose your desired number of questions.

The Number of Questions text box defaults to 30 and will only allow you to enter as many questions as there are total in the bank.

Next, you will select the desired CET Level, which in the case of a Pseudorandom exam will ensure that the first half of the exam is AT the chosen level and the second half is at or below the chosen level.

Finally, you will choose any Learning Objective filters that you desire. This is a multi-select list box and will allow you to choose as many choices as you see fit. The resulting exam will consist only of questions with one of the selected learning objectives. When there are no selections made, the exam will not be filtered at all.

Now you get to press the Generate Exam button.

You can press it as many times as you would like, each time it will run through the aforementioned algorithm and generate a unique exam. The right-hand side of the screen is a preview of the exam. Allowing you to verify that you are satisfied with the randomly generated result.

If it becomes necessary for you to change any of the settings, you can do that as well. Pressing the Generated Exam button simply readies the exam for download.

Once you are satisfied with your generated exam, you simply fill out the File Name text field with whatever you want and press the Download icon.

Once pressed, there will be no immediate indication of success or failure (which is something I will probably change in the future). But rest assured, it probably worked.

Once pressed, the exam will download to a file on your computer called Exam Downloads. If it does not already exist, it will be created and it will be placed on your Desktop.

Added to that folder will be two files:

  • fileName: the exam you just generated with all the CP1 formatting that we are used to.
  • fileName-Answer-Key: the answer key to said exam.
The Exam Downloads Folder.

Do not be alarmed by the unfamiliar file type. I will explain everything.

The unfamiliar file type you see here is an HTML file. This is because we have exported the exam as a custom made HTML file. This approach was MUCH more effective than anything else I tried. It prevents the weird formatting nuances of automatically converting it to a PDF (which I tried.)

You should be able to simply double click the file. The file will open in your default browser. For compatability reasons this works best in Microsoft Edge, if Edge is not your default you have to right click on the html file, select Open with, and choose Edge.

Once it is open in Edge, you can print it the way you would anything. Hopefully, the super-easy-to-use print preview window will appear allowing you to finely control the final output. Though, I have designed it in such a way that with all the defaults you do not have to change anything. Even with the margins set to something other than the defaults, your output should be normal looking.

Once you have printed it out the way you would anything else, you have the exam in your hand and all you had to do was press a few buttons.

Weekly Quiz

A periodic, random quiz that can be sent out to the whole team with the click of a button.

This is what the Send Weekly Quiz option does.

In the spirit of periodically brushing up on questions (see history lesson in the Philosophy Section), I have included the ability to send out what I am calling the Weekly Quiz

From the Home View, you will drop down the Hamburger Menu. From there you will simply select the Send Weekly Quiz option.

Simple, huh?

In beta, this only sends an email to yourself (so you can test it out). When fully implemented, it can be made to send an email to the CP1 alias. That email will consist of one or two randomly selected questions from the bank.

At the moment, it simply suggests that each shift should discuss it amongst themselves and send the answer to the training coordinators. However, I am certainly open to suggestions for what that email should say and how it should be handled. Once again, I can implement almost anything you can dream up, and would love to do so.

Question Editor

The ability to make rapid, straight-forward changes to the exam bank. Promoting regular management and updating. No clunky SharePoint interface to stifle your creative urges.

Had a particularly engaging discussing over a lunch break? Make a question about it. A big discussion came up during a drill? Make a question about it. You wake up from a fever dream about something that you wish EVERYONE was more familiar with?! MAKE A QUESTION ABOUT IT.

The goal of this app is to make editing, updating, adding, and deleting questions so seamless that you WANT to do it. Which will hopefully promote a more organic exam bank and prevent stagnation.

In the future, I want to add an automatic approval pipeline (if desired)

Making Changes to Existing Questions

First and foremost, this app is about database management made simple. The Home View should clue you in to this, as, the most apparent thing you can do is scroll through the entire bank.

From here you are free to search through and review all the questions and answers in the exam bank. You are also able to edit the details of each question.

You can do this via the Question Editor

You get here by pressing the right facing chevron on the Home View. For the question you wish to edit, of course.

From here, you can edit the question, the CET Level, the Learning Objective, and the Answer. As a reminder, the question text field will only allow you up to 255 Characters, you will get a warning otherwise. Additionally, the Question and Answer fields are required, so you at least have to put a placeholder if you wish to come back later.

You can also delete the individual questions from here if you wish. Fear not, you will be asked a very obvious "Are You Sure?" So, lest I made a mistake, you cannot accidentally delete a question.

Finally, you can abort back to the Home View via the Cancel Button. Pressing the Cancel Button will discard any changes you have not saved and leave the question unchanged, so you don’t have to worry about making any unintentional changes.

Philosophy

I don't want to imply that this isn’t a conversation that still needs to happen. I simply want to share my thoughts on what a training program like this should look like.

The Principles

It goes without saying that no training program was ever designed to NOT provide a reasonable level of certainty in an individual's skill level.

Some of us come from an industry in which the codes and credentialing are airtight, often written because many lives have been lost. Some of us come from the military, where rigorous training and compliance are paramount, or lives may be lost. That is how I know that we all understand the level of scrutiny a program like the CETTP is attempting to cultivate. So, we can all agree that this program should uphold all of us to the highest standard, especially a standard that our management can have faith in.

That is why I believe that we should hone the CET Training Program to reflect these beliefs. I believe that starts with the Question Bank, which is exactly why I spent a month creating this app and all the associated framework.

The Microsoft standard suggests that the exam bank should have a minimum of 100 questions and those questions should be periodically updated. The Question Editor and Add New Question features are designed to make this as simple as possible, and though I made a joke out of it in the earlier section, I believe we can take this one step further. Any time a training coordinator learns something new or an intense discussion during quals or a drill come up, it should be our standard practice to put that piece of information into circulation. This is the first principle that I think we should adopt. The desire to constantly improve our understanding and level of expertise.

The second principle is the logical successor to the first principle. We need to squash the tribal aspect of our knowledge. I have not been here for very long but the number of times I have heard the sentences "this is how we have always done it" or "I don’t think that is a very good idea" or "if we change it too many people will get confused" is too high. We need to adopt a growth mindset and ensure that any new or old knowledge is vetted and approved by the appropriate body of knowledge. Be it the documentation or the powers that be. That starts with updating and backing up all of our current knowledge with proof and (if possible) testing.

Another major paradigm shift that I believe would be beneficial is to hold ourselves to a much higher standard. We do not have the person-power to have the command and qualification hierarchy of larger facilities, and I am not even implying that we should have one. I do not bring this topic up because I want to bring people down, but because I want to bring everyone UP. If we are going to accept and adopt the intention behind Microsoft's data-driven, open-book policy then we should do our best to meet the intent rather than succumb to it. I suggest that we do this by limiting the resources that we can use during the test to what we would be okay with using during a procedure or a break-fix. We wouldn't rely on written notes or CET Study guides during a UPS failure, so why would we use them during the test. Furthermore, I believe that the standard by which the questions are written should also be higher. Not that they should take so long that you don't have time to think, or that they should be so difficult that you couldn't possibly know them without looking it up, but that they should be challenging enough to make us think and, if we don’t know the answer, know where to look it up. That being said, instead of fearing a more challenging exam, we should be constantly preparing for the exam. That way, we can be sure that those that succeed are well qualified, and those that fail can be helped in a meaningful way. That is why I included the Weekly Quiz feature.

That is all I have to say about that, and if you outright disagree with my thought process, I would love to hear your take on it all. If we all work together and combine our vastly different experiences and skillsets, the whole team will be better for it. And not just us, but our replacements.

Ebbinghaus and His Forgetting Curve

The Ebbinghaus forgetting curve is a graphical representation of the forgetting process. The curve demonstrates the declining rate at which information is lost if no particular effort is made to remember it.

The forgetting curve was defined in 1885 by German psychologist Hermann Ebbinghaus (1850-1909) in his book Memory: A Contribution to Experimental Psychology. He was the first psychologist who systematically studied memory and learning. Ebbinghaus recorded his findings mathematically in an attempt to discover patterns of forgetting and memory retention.

But as you can see, it is a natural and well-studied phenomenon. As such it should be treated like any other optimization problem that we encounter on a daily basis. The maths behind memory management are defined and we would be remiss to not take advantage of that.
